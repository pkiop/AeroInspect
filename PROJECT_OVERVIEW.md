# AeroInspect: 프로젝트 개요 및 핵심 개념 (Project Overview)

## 1. 프로젝트 목적
AeroInspect는 항공기 정비(MRO) 과정에서 발생할 수 있는 부품 누락, 오장착, 조립 불량 등을 AI가 실시간으로 식별하고 감항성(Airworthiness)을 검증하는 **다중 에이전트 기반 시각 점검 시스템(Multi-Agent Visual Inspection System)**입니다. 

## 2. 핵심 개념: AI 기반 자동 기준 모델 식별 및 매칭 (Auto-Baseline Matching)
기존의 수동 점검이나 단순 비교 시스템과 달리, AeroInspect의 핵심은 **"정비사가 촬영한 1장의 현장 사진만으로, AI가 스스로 어떤 기종의 어느 부위인지 식별하여 DB에서 완벽한 상태의 '정상 기준 이미지(Baseline)'를 알아서 찾아내어 대조하는 것"**입니다.

### 🔄 워크플로우 (Workflow)
1. **대상 촬영 (Live Inspection Capture)**
   - 현장의 정비사가 스마트폰이나 태블릿, AR 글래스 등을 활용하여 점검 대상 기체(또는 부품)를 1회 촬영합니다.
2. **AI 기체 및 부위 식별 (AI Identification)**
   - Vision AI가 촬영된 이미지를 분석하여 기종(Aircraft Model), 부위(Component Section), 각도(Angle)를 자동으로 추론합니다.
3. **기준 데이터베이스 자동 로드 (Database Retrieval)**
   - 식별된 메타데이터를 바탕으로 사내 공인 데이터베이스(Reference DB)에서 가장 완벽하게 조립된 상태의 정합 이미지(Baseline)와 부품 규격서(IPC)를 자동으로 불러옵니다. *(정비사가 기준 사진을 직접 찍거나 찾을 필요가 없습니다.)*
4. **다중 에이전트 비전 대조 (Multi-Agent Cross-Validation)**
   - 불러온 정상 기준 이미지와 현장 촬영 이미지를 픽셀 및 시맨틱 단위로 정밀 비교합니다.
   - 누락된 부품이나 미세한 균열, 규격과 다른 체결 상태 등을 찾아냅니다.
5. **무결성 검증 및 보고 (Integrity & Reporting)**
   - 탐지된 결함을 설계 규정(Deterministic Rules)과 교차 검증하고, 비행 가용 여부(GO/NO-GO)를 포함한 자동화된 감항 보고서를 생성합니다.

## 3. 데모 시스템에서의 구현 (현재 대시보드)
대시보드는 **단일 무대(Stage) 화면**으로 구성되어 있습니다. 시연자는 점검 대상 사진 **1장만** 등록하면 되고, 나머지(기종 식별 → 기준 형상 DB 조회 → 대조 → 판정)는 시스템이 이어서 진행합니다. 이는 위 "자동 매칭" 개념을 화면 흐름 그 자체로 표현하기 위한 것입니다.

### 🎬 시연 흐름
| 단계 | 화면 |
|---|---|
| 사전 준비 | 오퍼레이터 패널(`~` 키)에서 기준 형상 이미지를 미리 등록해 둔다 |
| 1. 투입 | 무대에 점검 대상 사진 1장을 드롭 |
| 2. 식별 | 스캔 오버레이와 함께 기종·부위·각도 칩이 순차 확정 |
| 3. 기준 매칭 | 기준 형상이 DB에서 착지하며 `MATCHED` 스탬프 |
| 4. 개시 | `점검 개시` 클릭 (또는 `Space`) — 시연 중 유일한 조작 |
| 5. 대조 | 4개 에이전트가 타임라인에서 순차 실행, 비교 와이프가 자동 스윕 |
| 6. LOCK-ON | 결함 영역에 바운딩 박스 슬램 |
| 7. 판정 | 전체화면 `NO-GO / HOLD / GO` 판정 |

### 🎛 오퍼레이터 패널 (`~` 키)
데모 조작부는 **전부 이 패널로 격리**되어 있으며 무대에는 노출되지 않습니다. 촬영 화면에 잡히면 시연의 설득력이 무너지기 때문입니다.

- **기준 형상 사전 로드** — DB 자동 매칭을 시뮬레이션합니다. *(시연 전 반드시 등록. 미등록 시 시스템이 패널을 자동으로 열어 알립니다.)*
- **식별 결과 문구** — 기종·부위·각도 텍스트를 시연 대상에 맞게 수정
- **추론 엔진** — Gemini API Key 입력 시 실제 멀티모달 대조를 수행하고, 비워 두면 내장 시뮬레이션으로 동작합니다. 실연동 실패 시 자동으로 시뮬레이션 폴백
- **검증 규칙** — 신뢰도 임계값, 방향 불일치 강제 플래그
- **연출** — 자동 진행 토글, 연출 속도(0.5×~2×)

> 판정 결과와 보고서는 데모 목적의 산출물이며, 보고서 하단에 그 취지가 명시되어 있습니다.
